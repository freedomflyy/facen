# Facen

基于人脸识别的课堂签到系统。项目面向校内课堂考勤场景，围绕“防代签、易部署、低延迟、可管理”几个核心目标实现了学生签到、教师发起签到、管理员统一管理，以及活体检测、人脸比对、地理围栏校验等功能。

这个仓库已经按公开展示场景做过整理：

- 不提交本地数据库、上传的人脸照片、IDE 配置、`node_modules`、实习报告和演示材料
- JWT 密钥和数据库路径支持通过环境变量覆盖
- 初始化演示数据已替换为通用示例账号，避免暴露具体个人或校内信息

## 项目背景

传统课堂签到方式普遍存在几个问题：

- 纸质签到和口头点名效率低，课堂时间占用明显
- 二维码或口令签到容易被代签、转发和作弊
- 教师统计和管理员追踪成本高，缺少统一留痕

这个项目的目标，是把“身份核验 + 活体检测 + 位置校验 + 考勤管理”整合到一个完整系统中，提升课堂考勤的真实性、自动化程度和管理效率。

## 核心功能

### 1. 学生端

- 账号注册与基础信息录入
- 摄像头拍照录入人脸
- 输入签到码验证签到会话
- 前端活体检测，要求完成随机动作
- 上传抓拍结果到后端执行人脸比对
- 查看个人历史签到记录

### 2. 教师端

- 创建签到会话
- 设置签到名称、时长、目标班级
- 支持带地理围栏的课堂签到
- 查看签到结果、缺勤情况和统计信息
- 支持手动补签
- 支持课堂人脸检测与人数统计

### 3. 管理端

- 教师管理
- 学生管理
- 班级管理
- 教师与班级分配
- 查看全部签到会话
- 查看和修正签到记录

## 系统亮点

### 1. 端云协同的双重活体检测

系统不是只做一次简单拍照比对，而是采用“前端配合式活体 + 后端静默式活体”的双层防护思路：

- 前端基于 `face-api.js` 做实时人脸检测和关键点跟踪
- 通过张嘴、点头、摇头等随机动作进行配合式活体验证
- 后端再使用 `MiniFASNetV2` 对上传图像做静默活体检测
- 双重校验能同时拦截照片、翻拍、屏幕展示等常见作弊方式

### 2. 高精度人脸识别校验

后端基于 `InsightFace` 完成人脸识别流程：

- 先做人脸检测
- 再提取人脸特征向量
- 最后通过相似度计算完成身份比对

相比只做图片上传保存的方案，这种方式更适合课堂签到这种“需要真实身份确认”的业务场景。

### 3. 地理围栏签到

系统支持签到位置校验：

- 学生端获取当前位置经纬度
- 教师创建签到时设置课堂中心位置和签到半径
- 后端通过球面距离公式判断是否位于允许范围内

这样可以减少远程代签，增强考勤结果的可信度。

### 4. 多角色协同管理

系统把课堂考勤拆分为三个角色：

- 学生负责签到执行
- 教师负责课堂签到管理
- 管理员负责全局数据维护和权限配置

这种设计使项目不只是一个算法演示，而是一个完整的业务系统。

## 关键流程

### 学生签到流程

1. 学生登录系统并输入签到码
2. 前端校验签到码对应的签到会话是否有效
3. 前端调用摄像头进行配合式活体检测
4. 活体通过后抓拍图像并提交后端
5. 后端依次执行会话校验、班级权限校验、地理围栏校验、静默活体检测、人脸比对、重复签到校验
6. 校验全部通过后写入签到记录，并自动判定正常或迟到状态

### 教师发布签到流程

1. 教师选择目标班级
2. 设置签到名称、时长、签到范围
3. 系统生成签到码和会话记录
4. 学生在有效时间内完成签到
5. 教师实时查看签到结果和未签到学生情况

## 快速开始

### 1. 启动后端

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python init_data.py
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 启动前端

```bash
cd frontend
npm install
npm run dev
```

默认前端开发服务会把 `/api` 代理到 `http://127.0.0.1:8000`。
